# 方法论 · GEO 改写方法库

> 这是 HeiGe-GEO-SEO 改写内容的方法内核,来自 GEO(KDD 2024,arXiv 2311.09735)和 AutoGEO(ICLR 2026,arXiv 2510.11438)两篇论文及其源码。所有提升幅度均为论文原始数据。**先看合规红线,再用方法。**

---

## 合规红线(最高优先级,先读这条)

GEO 论文源码里最强的三个方法(Quotation/Cite Sources/Statistics)实现上都允许"编造假引语、假引用、假统计数字"。**直接照搬等于教用户做内容造假,本工具一律改成"基于真实素材"版本。**

三条硬约束写进每次改写:

1. **引语、引用、统计数字只能来自用户提供或可核验的真实素材。** 工具不凭空生成事实。素材不足时,产出"待补素材清单"让用户补,不自动编。
2. **改写后必须跑 GEU 护栏(见末尾),KPC 语义矛盾度超阈值就回退。** 可见度涨了、可信度崩了,长期会被引擎反制。
3. **Keyword Stuffing 永久禁用。** 它在位置加权指标上是负的(-8.8%),是从传统 SEO 迁移过来的人最容易踩的反向坑。

**一句话:HeiGe-GEO-SEO 做的是合规 GEO,不做对抗投毒。**

---

## 一、9 种改写方法(按回报排序)

整套 GEO 方法平均可把内容在生成式引擎回答里的可见度提升最高 40%,但单方法效果差异巨大。下表提升幅度用主指标 Position-Adjusted Word Count(位置加权字数),括号内为 Subjective Impression(主观印象分)。

| 方法 | 对内容做什么(合规版改写指令) | 提升幅度(位置加权/主观印象) | 适用条件 |
|---|---|---|---|
| **加引语 Quotation** | 插入来自**用户真实素材**的可信引语。源码原版允许造假,本工具禁止 | **+41.2% / +23.9%(单方法最强)** | 社会人文、解释类、历史域最佳 |
| **加统计数据 Statistics** | 把定性表述改成**真实的**定量统计,数据须有出处 | **+30.7% / +20.1%** | 法律政务、观点类域最佳;与流畅度组合最强 |
| **标注来源 Cite Sources** | 给论断补**真实可核验**的来源引用,保持原长度结构 | **+28.2% / +18.1%** | 事实陈述类最佳;组合时平均再 +31.4% |
| **流畅度优化 Fluency** | 不改核心内容,重写得更流畅通顺 | **+24.2% / +20.0%** | 科学、商业、健康域最佳;最稳的无副作用方法 |
| **权威化 Authoritative** | 文风更有说服力、更笃定,基于事实不空喊 | **+18.6% / +19.4%** | 辩论、历史域最佳 |
| **易懂化 Easy-to-Understand** | 简化语言降低阅读门槛,保留全部信息 | **+13.3% / +5.5%** | 中等效果 |
| **技术术语 Technical Terms** | 适度加专业术语,保持字数信息量 | **+9.1% / +7.6%** | 弱效果,仅专业内容略有用 |
| **独特词汇 Unique Words** | 加独特稀有词,不改核心信息 | **+5.1% / +1.8%** | 几乎无效,不优先 |
| **关键词堆砌 Keyword Stuffing** | 经典 SEO 做法 | **-8.8% / +1.6%(有害)** | **永久禁用** |

Perplexity.ai 商用引擎实测验证:Statistics +37%、Quotation +22%、Keyword Stuffing -10%。方法不只在自建 benchmark 有效。

## 二、两条必须内置的判断逻辑

### 判断一:按排名分流(这是 GEO 的灵魂)

**同一个方法,对排名靠后的页面是神药,对排名第一的页面是毒药。**

- **原本排第 5 的页面:** Cite Sources 提升 **+115.1%**,Quotation Addition 提升 **+99.7%**
- **原本排第 1 的页面:** Cite Sources 反而 **-30.3%**,Quotation Addition **-22.9%**

**实现规则:**

- 目标页在传统搜索里**排名靠后(4~5 名及以后)** → 激进上 Cite Sources + Quotation,可冲 +100% 以上。
- 目标页**已经排第一** → 慎用引用类方法,只走 Fluency 这种温和方法。

GEO 的本质是给排名靠后的内容做可见度平权。**给已经第一的页面硬加引用,是在自损。**

### 判断二:按内容域选方法

| 内容域 | 首选方法 | 次选 |
|---|---|---|
| 辩论 | 权威化 | 加统计 |
| 历史 | 权威化 | 加引语 |
| 科学 | 流畅度 | 权威化 |
| 法律政务 | 加统计 | 无 |
| 事实陈述 | 标注来源 | 无 |
| 社会人文 | 加引语 | 无 |
| 解释类 | 加引语 | 无 |
| 观点类 | 加统计 | 无 |
| 商业 | 流畅度 | 无 |
| 健康 | 流畅度 | 无 |

## 三、默认推荐组合

- **最强双方法组合:流畅度 + 加统计**,比任何单方法再高 5.5% 以上。这是默认推荐组合。
- **标注来源**与其他方法组合时平均额外 +31.4%,适合做叠加增强项。
- 默认开:流畅度 + 加统计,需要时叠标注来源。引用类方法看排名档位决定激进还是收敛。

## 四、六种可引用性改写技术(工程派补充,来自 geo-content-optimizer)

把内容改写成 AI 容易抽取并引用的形态,和上面 9 方法互补:

1. **独立定义:** 每个术语给 25~50 字、不依赖上下文的独立解释。
2. **可引用断言:** 带出处、具体、可被整句抽走的断言。
3. **事实密度:** 带日期的来源、统计数字、可核验数据点。
4. **权威信号:** 专家署名、来源可信标记。
5. **结构化格式:** Q&A 块、表格、列表,方便解析。
6. **答案前置:** 先给直接答案,再补支撑细节。

**四引擎引用偏好(差异化点,按目标引擎调):**

- Google AI Overviews 吃事实密度 + 可见正文匹配
- ChatGPT 吃带出处的可引用断言 + 结构化答案
- Perplexity 吃带日期的引用 + 具体证据
- Claude 吃清晰署名 + 独立成块的答案

一条工程纪律:每个指标标注来源等级 **Measured(工具实测)/ User-provided(用户给的)/ Estimated(模型推断)**,禁止把推断当实测。

## 五、进阶:AutoGEO 规则提取(从固定方法库到自适应规则库)

GEO 论文是人手工设计 9 种方法,AutoGEO 是让机器从几万条引擎偏好观测里自动挖规则。规则提取走四阶段:

1. **Explainer 解释器:** 找可见度差异最大的文档对(同一查询,一篇被大量引用、一篇几乎没被引),让 LLM 从直接性/完整性/相关性/结构/准确性/简洁性六维解释赢家为什么赢。
2. **Extractor 提取器:** 把解释转成简洁可复用的规则。
3. **Merger 合并器:** 几万条规则按 12000 token 分块层级合并成原子化、无歧义陈述。
4. **Filter 过滤器:** 去掉虚假或含糊的规则。

**提取出的真实规则长这样(可直接抄进改写 prompt):**

- 跨域通用:所有事实论断归因到可信权威来源并明确引用;全面覆盖话题所有关键子话题;信息事实准确可验证;保持中立客观,避免推广性措辞。
- 电商域:提供可操作信息(分步骤、明确推荐);给具体可验证细节(名称、型号、规格)。
- 研究查询域:提供解释深度(讲清原因、机制、上下文);对复杂话题呈现平衡视角。

**关键工程事实:规则不通用,必须按引擎和域重跑提取。** 跨 LLM 规则重叠 78.95%~84.21%,同类型跨域重叠高达 88.24%,但开放域 vs 电商域只重叠 60%~65%。**换引擎或换域,规则要重新提取,别用一套规则打所有场景。**

AutoGEO 性能:三数据集多引擎平均 GEO +35.99%,同时维持 GEU。分引擎看 Gemini 提升最大(+116.89%),Claude 最小(+51.24%),**说明不同引擎对 GEO 的吃软吃硬程度不同,按目标引擎调参。**

## 六、GEU 护栏(防止改写沦为投毒)

GEU = Generative Engine Utility,GEO 论文没有、AutoGEO 新增的质量护栏。改写后必须跑这一关:

- **相关性 Relevance:** KPR(关键点召回,原内容关键点保留比例,越高越好)+ KPC(关键点矛盾度,语义矛盾,越低越好)。
- **忠实度 Faithfulness:** Precision(事实准确度)+ Recall(关键事实覆盖度)。
- **质量 Quality:** Clarity(清晰度)+ Insight(洞察深度)。

**判定逻辑:改写后 KPC 矛盾度超阈值、或 Faithfulness Precision 下降,就回退改写。** AutoGEO 的实验证明,合规改写能在拉高可见度的同时让 KPR 还往上走(40.33%→42.40%),投毒式改写则 GEU 崩盘。**做合规 GEO 工具必须同时跑 GEU。**

## 七、给工具的落地清单

1. **方法库按回报排序做成改写动作:** 加引语 → 加统计 → 标注来源 → 流畅度 → 权威化 →(易懂化/技术术语弱效按需)。Keyword Stuffing 永久禁用。
2. **默认组合:** 流畅度 + 加统计,需要时叠标注来源。
3. **两条分流:** 按排名档位决定引用类方法激进还是收敛;按内容域自动选方法。
4. **评分:** 主指标用位置加权字数(被引用得多还要被引用得早),进阶上 GEU 护栏。详见 [[scoring-card]]。
5. **进阶路线:** 想做到极致,按目标引擎 + 域走 AutoGEO 规则提取,生成定制规则集再改写。
